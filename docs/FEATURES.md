# DramaForge 功能设计

> 当前文档是 DramaForge 的唯一功能设计文档。旧的小云雀对标、聊天框模式、多 Pipeline、UI 像素级设计和旧 Spec 规划文档已废弃。

## 1. 产品定位

DramaForge 是一个开源的 AI 短剧生产系统，目标是把故事创意稳定推进到可审核、可编辑、可复用的短剧生产链路。

核心定位：

- 泛 Agent 首页作为登录后的工作入口。
- 通用聊天保留，用于创意探索、项目入口、快速问答和后续短剧任务引导。
- 短剧生产链路是核心差异化能力，不再把产品定位成泛视频生成器。
- 主链路聚焦：Story Bible -> 角色/场景资产 -> 分镜源码 -> 多模型成片。
- 首阶段验收目标是一部 5 集样片链路完整跑通。

目标用户：

- AI 短剧、漫剧、剧情号创作者。
- 网文/IP 团队，用于快速制作分镜样片、PV 和招商 demo。
- MCN 或营销团队，用于剧情化短视频广告生产。
- 需要私有化部署、多模型接入和成本控制的企业团队。
- 想二次开发 AI 视频生产链路的开源用户和开发者。

## 2. 核心生产链路

### 2.1 泛 Agent 首页 / 通用聊天

保留登录后首页和通用聊天能力。首页不再承载消费级会员售卖，也不展示没有真实后端支撑的功能承诺。

首页职责：

- 作为登录后默认跳转页面。
- 支持通用对话、创意探索和项目引导。
- 提供进入短剧工作台、项目列表、资产库、模型配置的入口。
- 将短剧主链路作为最重要的推荐动作。

### 2.2 Story Bible

Story Bible 是短剧项目的叙事基准，用于约束后续角色、场景、分镜和成片生成。

最小内容：

- premise：核心故事命题。
- world_rules：世界观和规则。
- character_relationships：人物关系。
- timeline：关键时间线。
- episode_arc：分集节奏和阶段目标。
- visual_style：视觉风格规则。
- continuity_notes：连续性注意事项。

Story Bible 可以由 AI 生成，也可以从上传剧本解析后补全。现有剧本字段如故事梗概、背景、设定和一句话故事继续保留，用于兼容旧项目和快速摘要。

### 2.3 角色/场景资产

角色和场景是可复用资产，不是一次性提示词。

角色资产应支持：

- 角色名、角色定位、人物小传、性格、关系。
- 音色描述。
- 基础形象图。
- 阶段形象，如前期、转折后、结局状态。
- 后续扩展三视图和表情/姿态参考。

场景资产应支持：

- 场景名、空间描述、内外景、时间状态。
- 参考图。
- 同一场景的多状态，如白天、夜晚、破损、庆典、回忆等。

### 2.4 分镜源码

分镜是 DramaForge 的核心可控层。每个镜头都应被当作可编辑源码，而不是不可解释的一次性生成结果。

镜头字段包括：

- 时长。
- 场景引用。
- 角色引用。
- 景别、角度、运镜。
- 角色动作、情绪、台词、旁白。
- 视觉参考。
- 图片提示词和视频提示词。
- 生成状态、失败原因和生成结果。

分镜源码必须能够在生成前审核，在生成失败后单镜头或单片段重试。

### 2.5 多模型成片

DramaForge 保留多模型配置和生成能力。模型能力按 chat、image、video、tts 等能力拆分管理。

多模型能力用于：

- 剧本和 Story Bible 生成。
- 角色/场景图片生成。
- 分镜图片生成。
- 镜头视频生成。
- 语音、旁白、字幕和合成。

系统应逐步提供成本统计和模型路由建议，但首阶段以可配置、可生成、可追踪为主。

## 6. MVP 验收标准

首阶段 MVP 以 5 集样片链路为验收。

必须完成：

- 用户登录后可以从泛 Agent 首页进入短剧工作台。
- 可以创建一个短剧项目。
- 可以上传剧本或由 AI 生成剧本。
- 可以形成 Story Bible。
- 可以生成角色和场景资产。
- 可以为至少一集生成可编辑分镜源码。
- 可以生成镜头或片段视频。
- 可以合成单集视频。
- 可以记录主要模型、耗时、成本或积分消耗。

不作为首阶段验收：

- 爆款复刻。
- 一镜到底。
- 泛长视频生成。
- 消费级订阅闭环。
- 完整团队协作。
- 自动发布到外部平台。

## 7. 企业接入保留项

以下能力继续保留，方便企业私有化和二次开发：

- 用户体系。
- 用户隔离。
- API Key 和模型供应商配置。
- 内置模型目录。
- 积分余额。
- 计费记录。
- 支付订单。
- 支付回调。
- 生成任务记录。

前端呈现原则：

- 计费和支付不作为首页营销内容。
- 企业接入配置应集中在设置页或管理页。
- 开源默认体验应先突出生产链路，而不是充值和套餐。
